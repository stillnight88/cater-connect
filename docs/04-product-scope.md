# Phase 4 — Product Scope

> Purpose: Define what the product will deliver in Version 1 and what it intentionally will not.

---

## 1. Core Product Outcome

Enable customers to confidently discover, book, and review local vendor services through a booking-verified trust system while providing a structured onboarding process for vendors.

---

## 2. Must-Have Features (MVP)

These features are required to deliver the core product outcome.

### Discovery

- Customers can search and view local vendors.
- Customers can browse services offered by a vendor.

### Transparency & Trust

- Customers can view reviews tied to completed bookings.
- Service ratings are calculated only from verified reviews.

### Vendor Onboarding

- Customers can apply to become vendors.
- Administrators can review vendor applications.
- Approved applicants become vendors.
- Rejected applications are tracked with a review outcome.

### Booking Flow

- Authenticated customers can create booking requests.
- Booking status is visible throughout its lifecycle.
- Booking lifecycle is tracked:

  `pending → accepted → rejected → completed → cancelled`

### Post-Booking

- Customers can submit one review per completed booking.

---

## 3. Nice-to-Have Features

These improve usability but are not required to deliver the core product value.

- Sorting and filtering of services
- Saved favorite vendors or services
- Booking history dashboard
- Notifications for booking updates

---

## 4. Explicitly Out of Scope (Version 1)

The following will **not** be implemented:

- Payment processing or refunds
- Real-time chat or messaging
- Mobile applications
- Advanced business analytics and reporting dashboards
- AI-based recommendations
- Dynamic bidding or auction systems
- Complex availability scheduling

---

## 5. Version Boundaries

Version 1 is complete when the platform supports the complete customer and vendor onboarding journey:

### Customer Journey

Discover Vendor

→ View Services

→ Create Booking

→ Track Booking

→ Submit Verified Review

### Vendor Journey

Customer Applies to Become Vendor

→ Administrator Reviews Application

→ Vendor Account Activated

→ Vendor Can Manage Services

Anything beyond these workflows is intentionally postponed for future versions.

---

## Phase Completion Rule

This phase is complete when:

- The core product outcome is clearly defined.
- MVP features are limited to the essential user journeys.
- Nice-to-have features are separated from the MVP.
- Version boundaries are explicit.
- Scope creep risks are minimized.
